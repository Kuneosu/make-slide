# 🎯 make-slide — AI 프레젠테이션 스킬 기획서

> AI 코딩 에이전트가 단일 HTML 파일로 발표 슬라이드를 생성하는 범용 스킬
> 대상: Claude Code, Gemini CLI, Codex 등 모든 AI 코딩 도구

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **이름** | make-slide |
| **결과물** | 단일 HTML 파일 (외부 의존성 최소화) |
| **타겟 유저** | AI 코딩 도구 사용자 (개발자, 발표자) |
| **배포** | GitHub 공개 레포 (kuneosu/make-slide) |
| **기본 폰트** | Pretendard (CDN) |
| **핵심 가치** | 토큰 절약 + 일관된 퀄리티 + 재사용 가능한 디자인 시스템 |

## 3. 입력 모드 (3가지)

### Mode A: 주제만 제공
```
"AI 트렌드에 대해 15분 발표 만들어줘"
```
→ AI가 구조 설계 → 슬라이드 + 대본 생성

### Mode B: 내용/자료 제공
```
"이 문서 기반으로 발표 만들어줘" + (문서/메모/데이터)
```
→ AI가 내용 정리 → 슬라이드 + 대본 생성

### Mode C: 대본 제공
```
"이 대본으로 슬라이드 만들어줘" + (발표 스크립트)
```
→ AI가 대본 분석 → 대본에 맞는 슬라이드 생성

## 4. 워크플로우

```
Step 1. 입력 분석
  └── 모드 판단 (A/B/C) + 발표 시간·장수·청중 파악

Step 2. 디자인 선택
  └── Vercel 갤러리 링크 제공 → 사용자가 테마 선택
  └── 선택 안 하면 용도에 맞는 테마 자동 추천

Step 2.5. 이미지 체크
  └── "슬라이드에 이미지를 삽입할 포인트를 체크할까요?" 사용자에게 질문
  └── Yes → 아웃라인에 이미지 위치 표시, URL 요청
  └── No → placeholder로 진행

Step 3. 구조 설계
  └── 슬라이드 아웃라인 생성 (제목 + 핵심 포인트)
  └── 사용자 확인/수정

Step 4. 생성
  └── GitHub 레포에서 선택된 테마의 reference.html + README.md fetch
  └── 레퍼런스 코드 기반으로 HTML 생성
  └── 대본 동시 생성 (슬라이드별 스피커 노트)

Step 5. 전달
  └── HTML 파일 저장
  └── 로컬 서버로 프리뷰 (옵션)
  └── 수정 요청 대응
```

## 5. HTML 슬라이드 필수 기능

### 네비게이션
- ← → 키보드 방향키로 페이지 전환
- 클릭/터치 네비게이션
- 슬라이드 번호 표시 (현재/전체)
- 프로그레스 바

### 전체화면
- F키 또는 버튼으로 풀스크린 토글
- ESC로 해제

### 스피커 노트 (발표자 모드)
- S키로 스피커 노트 패널 토글
- 각 슬라이드별 대본 내장
- 현재 슬라이드 + 다음 슬라이드 미리보기 (가능하면)

### PDF Export
- Ctrl+P 또는 전용 버튼으로 인쇄
- `@media print` CSS로 처리
- 인쇄 시 UI 컨트롤 (전체화면 버튼, 네비게이션, 프로그레스바 등) 자동 숨김
- 각 슬라이드 = 1페이지

### 반응형
- 16:9 기본 비율
- 다양한 해상도 대응

### 애니메이션
- 슬라이드 전환 효과 (fade/slide)
- 요소별 순차 등장 (stagger)

## 8. SKILL.md 핵심 구조

SKILL.md는 AI 에이전트가 읽고 따르는 지침서. 특정 플랫폼(OpenClaw 등)에 종속되지 않는 범용 포맷으로 작성.

```
# make-slide — AI Presentation Skill

## 개요
단일 HTML 파일로 발표 슬라이드를 생성하는 스킬.

## 사용 시점
사용자가 프레젠테이션, 슬라이드, 발표 제작을 요청할 때.

## 워크플로우
1. 입력 모드 판단 (주제 / 내용 / 대본)
2. 디자인 선택 (갤러리 링크 제공 또는 자동 추천)
3. 아웃라인 생성 → 사용자 확인
4. 테마 레퍼런스 fetch (GitHub raw URL)
5. HTML 생성 (레퍼런스 코드 기반)
6. 대본 생성 (슬라이드별)
7. 로컬 서버로 프리뷰 제공

## 테마 목록
(각 테마별 GitHub raw URL, 설명, 용도)

## HTML 필수 사양
(네비게이션, 전체화면, 스피커노트, PDF Export 등)

## 출력 규칙
- 단일 index.html (인라인 CSS/JS)
- 외부 의존성: 폰트 CDN만 허용 (Pretendard 필수)
- 대본은 HTML 내 data 속성 또는 별도 script.md 파일
```

## 9. Vercel 갤러리 페이지

- 단일 정적 HTML 페이지
- 10개 테마를 카드 형태로 나열
- 각 카드: 스크린샷 + 테마명 + 설명 + 용도 태그
- 카드 클릭 → 해당 테마의 라이브 데모 (reference.html)
- 반응형, 다크/라이트 지원
- 용도: AI가 사용자에게 링크를 주고 테마를 고르게 하는 인터페이스

## 10. 핵심 설계 원칙

1. **토큰 절약** — 매번 디자인을 생성하지 않고, 레퍼런스 코드를 fetch해서 참고
2. **단일 파일** — 결과물은 항상 하나의 HTML (인라인 CSS+JS)
3. **범용성** — 특정 AI 도구에 종속 X, SKILL.md만 읽으면 어디서든 동작
4. **확장성** — 테마 추가 = GitHub에 폴더 하나 추가
5. **품질 일관성** — 레퍼런스 코드가 품질 기준선 역할
6. **가벼운 결과물** — 불필요한 라이브러리 없이 바닐라 HTML/CSS/JS

## 12. 확정된 세부 결정사항

### 이미지 처리 — 하이브리드 방식
- 기본: placeholder (이모지, SVG 아이콘, CSS 도형)
- 사용자가 URL을 제공하면 `<img src>` 삽입
- **워크플로우 추가**: Step 2~3 사이에 "이미지 삽입 포인트를 체크할지" 사용자에게 먼저 질문
  - Yes → 아웃라인에 이미지 위치 표시, 사용자에게 이미지 URL 요청
  - No → placeholder로 진행

### 코드 하이라이팅 — 경량 라이브러리 CDN
- Prism.js (~2KB gzip) CDN 사용
- 허용 외부 의존성: 폰트 CDN + Prism.js CDN
- 코드 블록이 없는 발표에서는 로드하지 않음 (조건부 포함)

### core/ 인라인 전략 — A) reference.html에 전부 인라인
- 각 테마의 reference.html에 네비게이션/풀스크린/스피커노트/PDF 코드를 모두 포함
- AI는 reference.html 하나만 참고하면 됨 (토큰 효율 최대화)
- core/ 폴더는 원본 소스로만 유지, 테마 업데이트 시 빌드 스크립트로 주입
- 이유: AI가 여러 파일 fetch하는 복잡도를 없애고, 단일 참조점으로 품질 보장

### 다국어 — 언어 무관
- reference.html: 더미 텍스트 사용 (특정 언어에 종속 X)
- SKILL.md: 영어로 작성
- README.md: 한국어 + 영어 이중 문서
- 테마별 README.md: 한국어 + 영어 이중 문서
- SKILL.md에서 "사용자의 언어에 맞춰 콘텐츠를 생성하라"고 지시
